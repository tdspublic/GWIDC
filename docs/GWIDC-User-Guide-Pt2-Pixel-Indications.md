# GWIDC USER GUIDE PART 2:<br>PIXEL INDICATIONS DURING BOOT UP 

|Pixel Display|Description|
|:----------------------------------:|:----------------------------------------|
|Red pixel<br>@ position 0 ![Image of Neopixel with red pixel at top](images/GWIDC-1Redpixel.png)|Device is in WiFi Configuration Mode. It has opened the GWIDC_AP access portal and is waiting on user to supply network credentials through the portal.|
|Green pixel (flashing)<br>@ position 0 ![Image of Neopixel with green pixel at top](images/GWIDC-1Greenpixel.png)|Device is in WiFi Configuration Mode, has successfully connected to the network, and is about to reboot. |
|Yellow pixel<br>@ position 0 ![Image of Neopixel with yellow pixel at top](images/GWIDC-1Yellowpixel.png)|Part of normal boot up sequence. The device is initializing and attempting to connect to the network. <br><br>Connection may take a few seconds, but if the device appears to “hang” at this stage, there may be a problem with the network credentials. (This can happen if the device was rebooted from a “hung” Configuration State--see above. In this case unplug the device for a few seconds and then power it back up. If the device fails to connect, manually RESET the device and go back to the access portal to enter credentials).  <br><br>NOTE: NodeMCU WiFI radios can have issues if the device is physically too close to the router.|
|Blue pixel<br>@ position 0 ![Image of Neopixel with blue pixel at top](images/GWIDC-1Bluepixel.png)|Part of normal boot up sequence.  The device is connected to the network and ready to accept commands. <br><br>If the Hubitat IP address and port have been configured then this state will be very brief. If the Hubitat IP address and port are not configured (null or invalid values) the GWIDC will idle in this state.|
|Blue pixels<br>@ positions 0, 3, 7, and 11 ![Image of Neopixel with four blue pixels](images/GWIDC-4Bluepixels.png)|Part of normal boot up sequence. The device is connected to the network, has sent an initial STARTUP message to the Hubitat IP address/port, and is ready to accept commands from the Hubitat.<br><br>If the device stays in this state for an extended period:<br>- Verify that the device was supplied with the correct static IP and port number of the target Hubitat hub (check via web browser with `http://<GWIDC_IPAddress>/REPORT_SETTINGS` <br>- Verify that the Hubitat is online at the correct static IP address, and verify the device configuration (Device Network ID, Device Type, and device IP address).<br><br>If everything checks out, rebooting the device should fix the problem. Otherwise, the problem may be with the router or network itself|
|Yellow pixels<br>@ positions 0 and 6 ![Image of Neopixel with two yellow pixels at top](images/GWIDC-2Yellowpixels.png)|Device has disconnected from the network. It will attempt to reconnect with previously saved credentials.  This can take less than a second or up to a few seconds.|
|Any other pixel colors/patterns|The device pixels and the piezo buzzer change in response to properly formatted messages from the Hubitat and from browser HTTP calls|
